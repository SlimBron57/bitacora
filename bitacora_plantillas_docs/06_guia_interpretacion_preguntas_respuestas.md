# 06 · Guía para interpretar preguntas y respuestas

## Al preguntar
- Sé específico: di el **Topic** y el **Action** que quieres.
- Pide **contratos** y **artefactos** para validar avances.
- Solicita **diagramas** si el flujo no es obvio.

## Al responder (formato sugerido)
1. **Resumen (una línea)**
2. **Delta** (qué cambió y por qué)
3. **Artefactos** (rutas, PRs, docs)
4. **Decisiones** (y trade-offs)
5. **Trazabilidad** (Comando ↔ Servicio(s) ↔ Repo/Modelo)
6. **Estado y métricas**
7. **Riesgos**
8. **Next steps + Gate**

## Señales de alerta
- Mucha promesa y pocos artefactos.
- Saltos de estado sin pruebas (build/tests).
- Transversal “todopoderoso” (Spark) que acopla todo.
