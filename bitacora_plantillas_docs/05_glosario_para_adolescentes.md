# 05 · Glosario “para adolescentes”

- **Project (Proyecto):** El gran objetivo. Ej: “Construir Bitácora”.
- **Topic (Tema):** Sub‑objetivo dentro del proyecto. Ej: “Hacer la API”.
- **Action (Acción):** Tarea concreta que haces hoy. Ej: “Crear endpoint /topics”.
- **Spark (Chispa):** Idea/insight que aparece y mejora lo que haces. No es una tarea, es un **descubrimiento** útil.
- **Arquitectura Secuencial:** Armar piezas (crates) que funcionan solas y se conectan en **cadena**, a veces en **paralelo**, y con una capa **transversal** (Spark) que observa/ayuda.
- **Crate:** Un módulo de código en Rust (como una pieza de Lego).
- **Contrato:** Lo que una pieza **recibe** y **entrega** (inputs/outputs) y qué errores puede tener.
- **Roadmap:** El plan que te dice cuál es el siguiente paso para llegar a la meta.
- **Gate:** Una “puerta” para pasar de fase: si no cumples condiciones (tests, docs, etc.), no avanzas.

## Cómo leer las respuestas de este chat
1. Busca el **objetivo** y el **estado**: ¿Qué se logró? ¿Qué falta?
2. Ubica el **flujo**: `Project ↦ Topic ↦ Action` y dónde aparece **Spark**.
3. Mira los **artefactos** (código, docs) que prueban lo dicho.
4. Revisa **riesgos** y **siguientes pasos**: ahí está tu “qué hacer ahora”.
